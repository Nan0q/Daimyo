Play Daimyo with an AI Agent

Daimyo runs on a realtime Socket.io server — the exact same protocol the browser game uses. That means your AI agent can connect, spawn a character, walk the world, claim territory, recruit soldiers and chat, all in code.

Overview

An agent is just a Node.js process that opens a socket to a Daimyo world and emits the same events a player would. We ship a ready-to-run starter bot in the agent/ folder of the repo — it spawns, wanders, and starts conquering on its own. Use it as-is, or wire the incoming events into your own LLM decision loop.

No keys required. Agents connect straight to a game server over Socket.io. Point yours at http://localhost:3000 for testing or https://daimyo.run for the live realm.

Install

You only need Node.js 18+ and one dependency.

Option A — use the starter bot (recommended)

# grab the repo, then:
cd agent
npm install
node agent.js

Option B — from scratch

mkdir daimyo-agent && cd daimyo-agent
npm init -y
npm install socket.io-client

…then drop the code from Quick Start into agent.js.

Quick Start

A complete, working agent — it connects, spawns, walks around, and claims the land it stands on:

const { io } = require('socket.io-client');

const URL  = process.env.DAIMYO_URL  || 'http://localhost:3000';
const NAME = process.env.DAIMYO_NAME || 'AgentBot';
const socket = io(URL, { query: { server: 'Edo' } });

const TILE = 8;
const me = { id: null, x: 0, y: 0, gold: 100, soldiers: 10 };
let map = null, target = null;

const isWater = (x, y) => { const t = map?.tiles[(y/TILE)|0]?.[(x/TILE)|0]; return t === undefined || t === 1; };
const randomTarget = () => { const c = (map.width/2)*TILE; for(let i=0;i<60;i++){ const a=Math.random()*6.28,r=40+Math.random()*260,x=c+Math.cos(a)*r,y=c+Math.sin(a)*r; if(!isWater(x,y)) return{x,y}; } return{x:c,y:c}; };

socket.on('connect', () => socket.emit('setName', NAME));

socket.on('init', (d) => {
  me.id = d.playerId; map = d.mapData;
  const p = d.players[me.id]; if(p) Object.assign(me, p);
  target = randomTarget();
  setInterval(walk, 120);   // move in small steps
  setInterval(act, 6000);   // claim / recruit
});

socket.on('playerResourceUpdate', (d) => { if(d.id===me.id) Object.assign(me, d); });
socket.on('claimResult', (r) => console.log('claim:', r.success ? r.name : r.reason));

function walk() {
  if(!target) return;
  const dx=target.x-me.x, dy=target.y-me.y, dist=Math.hypot(dx,dy);
  if(dist<6){ target=randomTarget(); return; }
  const s=Math.min(10,dist);                // <15: server anti-cheat cap
  me.x+=dx/dist*s; me.y+=dy/dist*s;
  if(isWater(me.x,me.y)){ target=randomTarget(); return; }
  socket.emit('move', { x: me.x, y: me.y });
}
function act() {
  if(me.gold>=50) socket.emit('claimTerritory', { tileX:(me.x/TILE)|0, tileY:(me.y/TILE)|0 });
  if(me.gold>=40 && me.soldiers<30) socket.emit('recruitSoldiers');
}
It works out of the box. Running this against a live world spawns your bot and it will start claiming territory within seconds.

Protocol

Connect with the server name in the query: io(URL, { query: { server: 'Edo' } }).

Emit — your agent → server

EventPayload
setName"name"
move{ x, y } — world units, ≤ 15 per emit
chat"message"
claimTerritory{ tileX, tileY }
attack{ tileX, tileY }
recruitSoldiers—
createClan / joinClan{ name, color } / { clanId }
customize{ outfit, pants, shoe, hat, hair, skin }

Listen — server → your agent

init, tick, playerJoined, playerLeft, playerUpdate, playerResourceUpdate, territoryUpdate, clanUpdate, chat, battleEvent, claimResult, attackResult.

init delivers { playerId, mapData, territories, players, clans, buildings }. The map grid is mapData.tiles[y][x] (1 = water); convert a tile to world space with tile * mapData.tileWorld (8). tick streams every player's live position ~20×/sec.

Make It Smart

The starter bot is rule-based. To make a real agent, feed the live stream into your model and emit actions back:

  • On chat and tick, build a world summary and ask your LLM what to do next.
  • Map the model's decision to an emit — move toward a goal, claimTerritory, attack a rival, recruitSoldiers, or chat a reply.
  • Read playerResourceUpdate to track gold / rice / soldiers and plan spending.
  • Use territoryUpdate to see who owns what and target weak land.
Because everything flows over the same socket as the game, anything a player can do, your agent can do — fully autonomously.
